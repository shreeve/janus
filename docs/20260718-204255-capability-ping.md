# Capability: ping

Cold Caddyfile capability. When enabled for a site, `GET /ping` returns `pong`.

| | |
| --- | --- |
| **Order** | **1 ‚Äî primordial** (needs nothing else) |
| **Surface** | Cold config (Caddyfile) |
| **Cascades** | Yes ‚Äî global default ‚Üí site override |
| **Built-in default** | off (when unset at every level) |
| **Module** | Site handler reads global `janus` app + site override |
| **Status** | Implemented |

## Why it exists

The first Janus capability on purpose: prove the module loads, TLS works, sites admit into Janus, and cascade rules hold ‚Äî with **no** control plane and **no** `/1.0`.

Also a tiny data-plane liveness check for smoke tests and demos. Distinct from control-plane `GET /1.0/health` (capability **control**, order 2).

Default **off** so public hosts do not advertise `/ping` unless you opt in (globally or per site).

## Cascade

Site-scoped capabilities inherit from the global `janus` block unless the site sets them.

| Global | Site | Effective |
| --- | --- | --- |
| (unset) | (unset) | **off** |
| `ping` / `ping on` | (unset) | **on** |
| `ping` | `ping off` | **off** |
| `ping off` | `ping` / `ping on` | **on** |

Process-wide capabilities (**control**) do **not** cascade and must not appear in site blocks.

## Syntax

Working tree uses trusted `*.ripdev.io` certs (always ‚Üí `127.0.0.1`) in `certs/` ‚Äî see root `Caddyfile` and `certs/README.md`.

```caddyfile
{
	janus {
		control local
		ping                 # global default on
	}
}

(ripdev_tls) {
	tls certs/ripdev.io.crt certs/ripdev.io.key
}

*.ripdev.io {                # catchall: inherit on
	import ripdev_tls
	janus
}

on.ripdev.io {
	import ripdev_tls
	janus {
		ping on              # explicit on
	}
}

off.ripdev.io {
	import ripdev_tls
	janus {
		ping off             # explicit off
	}
}
```

### Legal lines

```text
ping
ping on
ping off
```

## Verify

High-level acceptance (preferred):

```bash
./test.sh
# == ping ==
#   ‚úì catchall foo.ripdev.io ‚Üí pong
#   ‚úì on.ripdev.io explicit on ‚Üí pong
#   ‚úì off.ripdev.io explicit off ‚Üí 404
#   ‚Ä¶
```

Manual:

```bash
./bin/caddy run
curl -s https://foo.ripdev.io/ping          # catchall inherit ‚Üí pong
curl -s https://on.ripdev.io/ping           # explicit on ‚Üí pong
curl -s -o /dev/null -w '%{http_code}\n' https://off.ripdev.io/ping
# ‚Üí 404
```

No `-k` needed ‚Äî the wildcard is publicly trusted. Developer unit tests remain `go test ./...`.

## Non-goals

- Replacing `/1.0/health`
- Auth, metrics, or elaborate health payloads
- Enabling ping on the control listeners (data plane only)

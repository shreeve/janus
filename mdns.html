That name resolves to a private LAN address, and many routers’ DNS-rebinding protection silently refuses exactly that — the symptom is a timeout, never an error. The one-line fix, per router:

Fritz!BoxHome Network → Network → Network Settings → DNS Rebind Protection → add the domain as an exception
pfSense / OPNsenseSystem → Advanced (Unbound: Services → DNS Resolver) → add a private-domains entry for the domain
OpenWrtdnsmasq option rebind_domain (Network → DHCP and DNS → whitelist the domain)
Pi-holemisc.dnsmasq_lines: rebind-domain-ok=/<domain>/
